# GitHub Project Setup

## Create one GitHub Project

Suggested views:

1. Board view
   - Backlog
   - This Week
   - In Progress
   - In Review
   - Blocked
   - Done

2. Table view
   - filter by week
   - group by skill area

3. Roadmap view
   - use weekly milestones across 8 weeks

## Recommended custom fields

- Week (1-8)
- Skill Area (HTML, CSS, JS, Accessibility, Git, Deployment, Datastar)
- Estimate (1, 2, 3, 5)
- Actual Hours
- Evidence Added (Yes/No)
- Deploy Link
- Review Status (Self, Manager, Revised)
- Outcome (Learning, Bugfix, Feature, Refactor, Demo)

## Milestones

- Week 1 Foundations
- Week 2 CSS
- Week 3 Responsive + A11y
- Week 4 JS Basics
- Week 5 API + State
- Week 6 Workflow
- Week 7 Specialization
- Week 8 Capstone

## Rules

- every task starts as an issue
- every issue belongs to a week and skill area
- every completed feature closes through a PR
- every Friday ends with a retrospective issue and manager update

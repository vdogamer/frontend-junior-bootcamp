# GitHub Project Setup
## 4-Week Front-End Sprint with Bootstrap 4.4.1

## Recommended project views

### 1. Board view
Use these columns:
- Backlog
- This Week
- In Progress
- In Review
- Blocked
- Done

### 2. Table view
Use this to sort and filter by:
- Week
- Skill Area
- Review Status
- Evidence Added

### 3. Optional roadmap view
Use weekly target dates for:
- Week 1 Foundations
- Week 2 Responsive UI
- Week 3 JavaScript App
- Week 4 Capstone
- Buffer / Review Prep

## Recommended custom fields

- **Week:** 1, 2, 3, 4, Buffer
- **Skill Area:** HTML, CSS, Bootstrap 4.4.1, JavaScript, Accessibility, GitHub, Deployment, Docs
- **Estimate:** 1, 2, 3, 5
- **Actual Hours**
- **Evidence Added:** Yes / No
- **Deploy Link**
- **Review Status:** Self, Manager, Revised
- **Demo Worthy:** Yes / No

## Milestones

- Week 1 Foundations + Bootstrap setup
- Week 2 Responsive layout + accessibility
- Week 3 JavaScript app
- Week 4 Capstone + manager package
- Buffer / Review prep

## Rules

- every task starts as an issue
- every issue gets a week and skill area
- every completed task closes through a PR when possible
- every Friday ends with a deploy, retrospective, and evidence update
- every UI-heavy PR includes screenshots
- every week must end with something demoable

## Recommended starter issues

### Week 1
- [BUILD] Set up repo, README, and GitHub Project
- [BUILD] Add Bootstrap 4.4.1 starter template
- [BUILD] Create semantic landing page structure
- [BUILD] Deploy first version

### Week 2
- [BUILD] Add responsive navbar and card layout
- [BUILD] Improve spacing, typography, and custom CSS
- [BUILD] Fix accessibility and responsive issues

### Week 3
- [BUILD] Create app skeleton for Faculty Directory Lite or dashboard
- [BUILD] Add search and filter
- [BUILD] Add form handling and localStorage or JSON rendering

### Week 4
- [BUILD] Add loading / empty / error states
- [BUILD] Improve README and screenshots
- [BUILD] Final capstone cleanup and demo prep

## What your manager should be able to see quickly

- a clean board with work moving to Done
- weekly retrospectives
- links to deployed projects
- screenshots in pull requests
- an evidence log showing growth over time

# 8-Week Curriculum

## Assumptions

- 10 to 12 focused hours per week
- goal = junior front-end readiness, not framework mastery
- primary stack = HTML, CSS, JavaScript, Git/GitHub, accessibility, deployment
- optional company-aligned track = Datastar in Week 7

## Success outcomes by the end of Week 8

- build and deploy 3 portfolio-quality front-end projects
- use semantic HTML and accessible forms
- create responsive layouts with Flexbox and Grid
- manipulate the DOM with JavaScript and handle events
- fetch and render API data
- store client state with localStorage
- use GitHub Issues, Projects, branches, PRs, and releases
- document progress clearly enough for manager review

## Week 1 - HTML foundations + GitHub workflow

### Learn
- semantic HTML
- document structure
- links, lists, images
- forms basics
- Git basics: clone, add, commit, push, branch

### Build
- personal profile page or company-style landing page

### GitHub evidence
- issue for each section
- PR with screenshot
- deployed page
- weekly retrospective

### Exit criteria
- valid page structure
- accessible labels on form fields
- clean README for project

## Week 2 - CSS fundamentals

### Learn
- selectors
- box model
- spacing
- typography
- colors
- pseudo-classes
- CSS variables

### Build
- restyle Week 1 page into a polished responsive page

### GitHub evidence
- before/after screenshots
- issue for responsive fixes
- PR showing CSS organization

### Exit criteria
- styles separated cleanly
- mobile and desktop layouts work
- can explain specificity and box model

## Week 3 - Layout + responsive design + accessibility

### Learn
- Flexbox
- Grid
- media queries
- responsive images
- semantic controls
- keyboard navigation basics
- accessibility testing basics

### Build
- marketing page with hero, features, pricing, contact form
- include tablet and mobile breakpoints

### GitHub evidence
- Lighthouse screenshot
- accessibility checklist
- PR notes on keyboard/focus behavior

### Exit criteria
- responsive across common widths
- no obvious accessibility blockers
- layout built with Flexbox/Grid, not hacks

## Week 4 - JavaScript fundamentals

### Learn
- variables
- functions
- conditionals
- loops
- arrays and objects
- events
- DOM selection and updates

### Build
- interactive to-do list, habit tracker, or FAQ accordion

### GitHub evidence
- issues split by feature
- PR with test steps
- short demo video link

### Exit criteria
- app responds to user interaction
- code broken into small functions
- no inline JS except where absolutely needed

## Week 5 - JavaScript in real UI work

### Learn
- form validation
- localStorage
- JSON
- fetch
- async/await
- error handling

### Build
- small dashboard or search app powered by a public API
- persist user preferences or saved items locally

### GitHub evidence
- bug issue for API/error states
- PR with loading/error/success screenshots
- deployment link

### Exit criteria
- handles loading and error states
- persists some state locally
- code readable and commented where needed

## Week 6 - Professional workflow

### Learn
- npm scripts basics
- linting/formatting
- deployment workflow
- Lighthouse interpretation
- README writing
- debugging in browser devtools

### Build
- refactor one earlier project to improve code quality
- add polish: empty states, focus states, loading states

### GitHub evidence
- PR titled "refactor"
- issue checklist completed
- Lighthouse score log

### Exit criteria
- repo feels professional
- project can be deployed repeatedly
- manager can review work without asking where anything is

## Week 7 - Company-aligned specialization

### Recommended Track A: Datastar familiarity
- understand hypermedia-first UI thinking
- build one tiny interaction using Datastar
- use HTML-first patterns and `data-*` attributes

### Optional Track B: Resume-first framework exposure
- learn basic component thinking in React or similar framework
- rebuild one Week 4 or Week 5 feature as a componentized app

### Build
- one focused mini-project, not a giant app

### GitHub evidence
- architecture notes
- tradeoff notes
- PR comparing vanilla vs chosen framework approach

### Exit criteria
- can explain why you chose the approach
- can demo the project in under 3 minutes

## Week 8 - Capstone + manager demo

### Build
- final capstone tied to real work value:
  - company landing page section redesign
  - internal tool front-end prototype
  - API dashboard
  - accessible form-heavy workflow

### Deliver
- deployed production-style URL
- polished README
- architecture note
- Lighthouse results
- 3-minute demo recording
- final retrospective

### Exit criteria
- looks professional
- works on mobile
- handles basic errors
- accessible navigation/form behavior
- GitHub history shows disciplined workflow

## Daily cadence suggestion

- 60 min learning
- 60-90 min building
- 15 min commit + update issue
- Friday: 30 min review and retrospective

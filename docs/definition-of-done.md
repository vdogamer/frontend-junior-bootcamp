# Definition of Done

A task is done only when:

- code is committed to a feature branch
- pull request is opened
- screenshot or demo is attached for UI changes
- linked issue is updated or closed
- feature works on mobile and desktop
- obvious console errors are resolved
- accessibility basics checked:
  - semantic elements
  - labels on form fields
  - visible focus state
  - keyboard reachability where relevant
- deployment is updated if user-visible
- README or notes updated if behavior changed
